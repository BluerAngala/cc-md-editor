
## 三、逐个源详细说明

### 1. 中国政府网·最新政策（gov-policy）

**ID**: `gov-policy`  
**来源**: https://www.gov.cn/zhengce/zuixin/

#### 请求

```
GET https://www.gov.cn/zhengce/zuixin/ZUIXINZHENGCE.json
Headers: User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
Timeout: 30s
编码: utf-8
```

直接返回 JSON 数组，无需额外解析。这是所有源中数据质量最高的——字段完整、结构化、无 HTML 清洗负担。

#### 数据结构（列表项）

| 字段 | 来源 | 说明 |
|---|---|---|
| `TITLE` | JSON | 政策标题 |
| `DOCRELPUBTIME` | JSON | 发布时间，如 `2026-06-28 16:30:00` |
| `URL` | JSON | 政策详情页链接 |
| `SUB_TITLE` | JSON | 副标题（用作 brief 摘要） |

#### 数据清洗

- **日期过滤**：`datetime.strptime(date_str[:10], '%Y-%m-%d')`，与 `datetime.now() - timedelta(days=days)` 比较
- **标题**：直接取 `TITLE` 字段，无需清洗
- **URL**：直接取 `URL` 字段
- **文章 ID**：从 URL 中正则提取 `content_(\d+)\.htm`

#### 文章抓取（fetch_article）

```
GET {URL}
Headers: User-Agent: Mozilla/5.0...
Timeout: 30s
```

**正文提取**：依次尝试 3 个 CSS 选择器：`id="UCAP-CONTENT"`、`class="content"`、`class="article"`，取第一个匹配且内容 >200 字符的。

**HTML 清洗管线**：
1. 移除 `<script>` 和 `<style>` 标签及其内容
2. `<br>` → 换行，`</p>` / `</div>` / `</li>` → 双换行（段落分隔）
3. 移除所有 HTML 标签
4. 实体解码：`&nbsp;` → 空格、`&ldquo;` → `"`、`&rdquo;` → `"`、`&mdash;` → `—`、`&hellip;` → `…`
5. 按行去空白行、trim

**图片提取**：`<img src="...">` → 只保留 `http` 开头的绝对 URL

### 2. 央视网（cctv-law）

**ID**: `cctv-law`  
**来源**: https://news.cctv.com/ 法治频道

#### 请求

```
GET https://news.cctv.com/2019/07/gaiban/cmsdatainterface/page/law_1.jsonp?cb=law
Headers: X-Requested-With: XMLHttpRequest
Timeout: 30s
```

返回 JSONP 格式：`law({...})`，需先剥离外层 `law(` 和末尾 `)` 再 `json.loads()`。

#### 数据结构（列表项）

JSON 路径：`data.list[]`

| 字段 | 来源 | 说明 |
|---|---|---|
| `title` | JSON | 标题 |
| `focus_date` | JSON | 日期，如 `2026-06-28 10:30:00` |
| `url` | JSON | 文章链接 |
| `brief` | JSON | 摘要 |
| `image` / `image2` / `image3` | JSON | 配图 URL（最多 3 张） |
| `keywords` | JSON | 关键词数组（空格分隔的字符串，需 `.split()`） |

**特点**：自带摘要和配图，数据质量高，无需额外抓取正文就能做简报。

#### 数据清洗

- **关键词**：`i['keywords'].split()` —— JSON 中是空格分隔的字符串，需拆分为数组
- **日期过滤**：`datetime.strptime(i['focus_date'][:10], '%Y-%m-%d')`
- **文章 ID**：从 URL 正则提取 `ARTI[a-zA-Z0-9_-]+`

#### 文章抓取（fetch_article）

```
GET {URL}
Headers: User-Agent: Mozilla/5.0
Timeout: 30s
```

**特殊处理**：央视网文章正文存在 JS 变量 `var contentdate = '...'` 中，是转义后的 HTML 字符串。

1. 定位 `var contentdate` 在 HTML 中的位置
2. 找到字符串的起始引号（`'` 或 `"`）
3. 手动解析转义字符串（处理 `\n`、`\/`、`\"`、`\'`）
4. 得到原始 HTML 后，走标准清洗管线

**日期提取**：先尝试页面内正则匹配，失败后从 URL 路径 `/YYYY/MM/DD/` 提取。

**HTML 清洗管线**（同政府网，额外处理）：
- `&quot;` → `"`、`&lt;` → `<`、`&gt;` → `>`
- 残留 `&xxx;` 实体直接删除

## 五、缓存架构

```
cache/
├── raw/                    # 原始列表数据
│   ├── cctv-law/
│   │   └── 2026-07-01.json  # {source, count, items: [...]}
│   ├── gov-policy/
│   │   └── 2026-07-01.json
│   └── ...
├── articles/               # 单篇文章缓存
│   ├── ARTIxxx.json        # 结构化元数据 + 正文
│   ├── ARTIxxx.md          # Markdown 格式全文
│   └── ...
└── exports/                # 汇编输出
    └── 2026-07-01_法律资讯简报.md
```

- **raw**：按源+日期分文件，每天覆盖更新
- **articles**：按文章 ID 分文件，永久缓存，命中后不再重复抓取
- **exports**：每次 compile 生成新文件

## 六、通用 HTML 清洗管线

所有 HTML 抓取源共用同一套清洗逻辑（在各自 `fetch_article()` 中实现）：

```
原始 HTML
  ↓ 1. 移除 <script>...</script>（含内容）
  ↓ 2. 移除 <style>...</style>（含内容）
  ↓ 3. <br> → \n
  ↓ 4. </p> / </div> / </li> → \n\n（段落分隔）
  ↓ 5. 移除所有 HTML 标签
  ↓ 6. 实体解码：
       &nbsp; → 空格
       &ldquo; → "
       &rdquo; → "
       &mdash; → —
       &hellip; → …
       &amp;  → &
       &quot; → "
       &lt;   → <
       &gt;   → >
  ↓ 7. 按行去空白行、trim
  ↓ 8. 双换行连接
  → 纯文本正文
```
